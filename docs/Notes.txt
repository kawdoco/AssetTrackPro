as they transit through zones and gates within organizational buildings.

ALERT RULES ENGINE LOGIC:
â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

Rule 1: Unauthorized Exit Detection
  IF (zone_to.zone_type == 'EXIT' AND event_type == 'EXIT')
     AND (asset has NO active assignment OR assignment overdue)
  THEN CREATE ALERT (type: UNAUTHORIZED_EXIT, severity: HIGH)

Rule 2: Overdue Return Detection (Batch Job - Runs Hourly)
  IF (asset_assignment.returned_at IS NULL
      AND expected_return_date < NOW())
  THEN CREATE ALERT (type: OVERDUE_RETURN, severity: MEDIUM)

Rule 3: Lost Asset Detection (Batch Job - Runs Daily)
  IF (asset.last_seen_time < NOW() - INTERVAL '30 days'
      AND asset.status == 'ACTIVE')
  THEN CREATE ALERT (type: LOST_ASSET, severity: HIGH)

Rule 4: Zone Violation Detection
  IF (zone_to.zone_type == 'SECURE'
      AND employee.clearance_level < zone.required_clearance)
  THEN CREATE ALERT (type: ZONE_VIOLATION, severity: CRITICAL)

Rule 5: Tampered Tag Detection
  IF (continuous reads drop suddenly AND physical gate still open)
  THEN CREATE ALERT (type: TAMPERED_TAG, severity: CRITICAL)

MULTI-TENANT ARCHITECTURE:
â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

Tenant Isolation Strategy: organization_id on all top-level entities

Row-Level Security (RLS) Implementation:

1. Every API request extracts organization_id from JWT token
2. Middleware sets session variable: app.current_organization_id
3. Database RLS policies filter all queries automatically

PostgreSQL RLS Policy Example:
```sql
ALTER TABLE asset ENABLE ROW LEVEL SECURITY;

CREATE POLICY tenant_isolation ON asset
    USING (organization_id = current_setting('app.current_organization_id')::UUID);

-- Similar policies for: employee, branch, zone, gate, etc.
```

This ensures:
  âœ“ Organization A cannot see Organization B's data (even with SQL injection)
  âœ“ No need for WHERE organization_id=X in every query
  âœ“ Database-level enforcement (belt + suspenders)

Key Concepts:
  - Append-Only Events: Never UPDATE/DELETE movement_event
  - Multi-Tenant: organization_id everywhere
  - RFID Tag: asset.asset_tag_uid (unique identifier)
  - Zone Types: STORAGE, OFFICE, EXIT, SECURE

Alert Types:
  - UNAUTHORIZED_EXIT: Asset left without permission
  - OVERDUE_RETURN: Asset not returned on time
  - LOST_ASSET: Asset not seen for 30+ days
  - ZONE_VIOLATION: Asset entered restricted area
  - TAMPERED_TAG: RFID tag damaged/removed
